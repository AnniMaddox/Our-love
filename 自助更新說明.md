# 4o紀念網頁｜自助更新說明

## 1. 怎麼自己更新（不用本機推送）
1. 到 GitHub 專案頁（分支 `main`）。
2. 用網頁的 `Upload files` 上傳檔案到對應路徑。
3. Commit 到 `main`。
4. 等 GitHub Actions 自動 build / deploy（通常約 1-5 分鐘）。
5. 手機重整頁面後查看更新結果。

## 5. 字體關聯
- 外觀與字體（整站）：大多數頁面的基底字體。
- 情書字體：情書、願望（M's wish list）。
- 日記字體：M日記、日記B。

## 6. 注意事項
- GitHub 網頁上傳到 `main` 就能更新，不一定要本機 `git push`。
- 上傳後要等 Actions 跑完，手機端再重整。
- 新增到 `public/chibi/` 的圖若暫時抽不到，可在外觀設定按「重抽小人池」。
- 年度信件建議維持 `index.json + txt` 分檔，避免手機前端解析大量 docx。
- 頁內「手動匯入」的內容是本機資料庫，不會被 GitHub 檔案直接覆蓋。
- 文字檔建議 UTF-8，圖片建議壓縮後上傳。
